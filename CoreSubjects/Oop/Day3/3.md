# üß¨ Inheritance in Java (With Simple Theory + Examples)

---

## ‚úÖ What is Inheritance?

> **Inheritance** is the process where **one class (child/subclass)** inherits the **properties (variables) and behaviors (methods)** of **another class (parent/superclass)**.

‚úÖ It helps **reuse code** and build a **hierarchy** of classes.

---

## üîπ Why Use Inheritance?

1. ‚úÖ **Code Reusability** ‚Äì Write once, use many times
2. ‚úÖ **Extensibility** ‚Äì Build on top of existing code
3. ‚úÖ **Organized Structure** ‚Äì Parent-child relationships are clear
4. ‚úÖ **Polymorphism** ‚Äì Enables method overriding for flexibility

---

## üî∏ Syntax

```java
class Parent {
    // parent class members
}

class Child extends Parent {
    // child class inherits from Parent
}
```

üîπ `extends` keyword is used to inherit from a class.

---

## üë®‚Äçüë©‚Äçüëß Real-World Example

* **Vehicle** (Parent) ‚Üí has `speed`, `start()`, `stop()`
* **Car** (Child) ‚Üí inherits everything from Vehicle, and adds `airConditioning` or `musicPlayer`

---

## üîç Simple Code Example

```java
// Parent class
class Animal {
    void eat() {
        System.out.println("This animal eats food");
    }
}

// Child class
class Dog extends Animal {
    void bark() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat();   // inherited method
        d.bark();  // own method
    }
}
```

### üü¢ Output:

```
This animal eats food  
Dog barks
```

## üîÑ Types of Inheritance in Java

| ---------------- | ------------------ | --------------------------------------------- |

| **Multilevel**   | ‚úÖ Yes              | Chain of inheritance (A ‚Üí B ‚Üí C)              |
| **Hierarchical** | ‚úÖ Yes              | One parent, multiple children                 |

## üîß Method Overriding (Important in Inheritance)

Child class can **override** a parent class method (same name + parameters):

```java
class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}
```

---

## üì¶ `super` Keyword

Used to:

1. Access parent class methods/variables
2. Call parent class constructor

```java

    Animal() {
        System.out.println("Animal Constructor");
    }
}

class Dog extends Animal {
    Dog() {
        super();  // calls parent constructor
        System.out.println("Dog Constructor");
    }
}
```

* Java doesn‚Äôt support **multiple inheritance** using classes to avoid ambiguity (diamond problem).
* Interfaces can be used for **multiple inheritance**.

# üîê `private` Keyword in Java (Simplified with Examples)

---

## ‚úÖ What is `private`?

> `private` is an **access modifier** in Java used to **restrict access** to a class member (variable, method, or constructor).

üîí If something is marked `private`, it can **only be accessed within the same class**.

---

## üîë Why Use `private`?

* To implement **encapsulation** (data hiding)
* To **protect** variables and logic from being accessed or modified directly
* To **control access** using `getter` and `setter` methods

---

## üî∏ Where Can You Use `private`?

| Can be applied to | Meaning                                                           |
| ----------------- | ----------------------------------------------------------------- |
| **Variables**     | Only class methods can access it                                  |
| **Methods**       | Cannot be called outside the class                                |
| **Constructors**  | Used in Singleton or Factory patterns to restrict object creation |
| **Inner Classes** | Can be private to restrict scope                                  |

---

## üîç Example ‚Äì Private Variable

```java
class Student {
    private int age; // üëà private variable

    // public method to access private variable
    public void setAge(int a) {
        if (a >= 0) {
            age = a;
        }
    }

    public int getAge() {
        return age;
    }
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student();
        s.setAge(20);
        System.out.println("Age: " + s.getAge());  // ‚úÖ Age: 20
        // s.age = 25;  ‚ùå Error: age has private access
    }
}
```

### üü¢ Output:

```
Age: 20
```

---

## üîí Trying to access private directly

```java
class Demo {
    private int num = 10;
}

public class Test {
    public static void main(String[] args) {
        Demo d = new Demo();
        System.out.println(d.num);  // ‚ùå Error: num has private access
    }
}
```

---

## üë®‚Äçüè´ Real-World Analogy

Think of a **Bank ATM Machine**:

* You can **view balance** or **withdraw cash** using buttons (public methods).
* You **cannot access the internal balance variable** or machine logic ‚Äî it‚Äôs `private`.

---

## üìå Key Points

| Point                        | Explanation                               |
| ---------------------------- | ----------------------------------------- |
| Scope                        | Only accessible **within the same class** |
| Helps in                     | **Encapsulation** (protecting data)       |
| Access outside class?        | ‚ùå Not allowed directly                    |
| Access via `getter`/`setter` | ‚úÖ Yes, using public methods               |

---

## üîÅ Common Pairing

| Access Modifier         | Scope                      |
| ----------------------- | -------------------------- |
| `private`               | Same class                 |
| `default` (no modifier) | Same package               |
| `protected`             | Same package + subclass    |
| `public`                | Accessible from everywhere |

---

## ‚ùìCan `private` methods be inherited?

No, **private methods are not inherited** by subclasses and **cannot be overridden**.

# üß≠ `super` Keyword in Java ‚Äì Simplified Notes with Examples

---

## ‚úÖ What is `super`?

The `super` keyword is used to **refer to the immediate parent class** in Java.

It is mainly used for:

1. Accessing **parent class constructor**
2. Accessing **parent class method**
3. Accessing **parent class variable**

---

## üî∏ When to Use `super`?

* When child and parent classes have the **same method or variable names**, and you want to call the **parent's version**.
* To call the **parent class constructor** explicitly.
* To **avoid code duplication** by reusing parent behavior.

---

## üîπ 1. Using `super` to Call Parent Class Method

```java
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    void sound() {
        super.sound(); // üëà calling parent method
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.sound();
    }
}
```

### ‚úÖ Output:

```
Animal makes a sound
Dog barks
```

---

## üîπ 2. Using `super` to Call Parent Class Variable

```java
class Vehicle {
    int speed = 60;
}

class Car extends Vehicle {
    int speed = 100;

    void display() {
        System.out.println("Child speed: " + speed);
        System.out.println("Parent speed: " + super.speed);  // üëà accessing parent variable
    }
}
```

### ‚úÖ Output:

```
Child speed: 100
Parent speed: 60
```

---

## üîπ 3. Using `super()` to Call Parent Constructor

```java
class Person {
    Person() {
        System.out.println("Person constructor");
    }
}

class Student extends Person {
    Student() {
        super(); // üëà calling parent constructor
        System.out.println("Student constructor");
    }
}
```

### ‚úÖ Output:

```
Person constructor
Student constructor
```

---

## ‚ö†Ô∏è Note:

* If you don‚Äôt explicitly write `super()`, Java automatically adds it as the **first line** of the subclass constructor.
* If the parent class does not have a **no-argument constructor**, then **you must** use `super(args...)` with parameters.

---

## üß† Real-World Analogy

> Think of a subclass as a new version of something old.
> You can still **access old features** using `super`.

For example, a child inherits behavior from a parent. If both have a "speak()" method, the child can still call the parent‚Äôs version using `super.speak()`.

---

## üßæ Summary Table

| Use Case                | Example Syntax             | Purpose                       |
| ----------------------- | -------------------------- | ----------------------------- |
| Call parent variable    | `super.varName`            | Access hidden field in parent |
| Call parent method      | `super.methodName()`       | Reuse parent behavior         |
| Call parent constructor | `super()` or `super(args)` | Initialize parent constructor |

## üîπ What is Single Inheritance?

**Single Inheritance** means:

> One class inherits (extends) from only **one parent class**.

It's like a **child inherits features from one parent**.

---

### ‚úÖ Real-Life Example:

* **Parent**: Father
* **Child**: Son

The Son inherits the properties or behavior from Father ‚Äî like surname, habits, etc.

---

### ‚úÖ Java Code Example:

```java
// Parent class
class Father {
    void display() {
        System.out.println("I am the Father.");
    }
}

// Child class (inherits Father)
class Son extends Father {
    void show() {
        System.out.println("I am the Son.");
    }
}

// Main class to run the code
public class Main {
    public static void main(String[] args) {
        Son s1 = new Son();

        s1.display();  // From Father class
        s1.show();     // From Son class
    }
}
```

---

### üìå Output:

```
I am the Father.
I am the Son.
```

---

### ‚úÖ Diagram to Understand (Text form):

```
Father class
   ‚Üë
   |
Son class
```

* `Son` **extends** `Father`
* `Son` can use both `show()` and `display()` methods
* But `Father` can't use `show()` (because parent doesn't inherit from child)

---

| Term               | Meaning                                   |
| ------------------ | ----------------------------------------- |
| Parent Class       | The class being inherited from (`Father`) |
| Child Class        | The class that inherits (`Son`)           |
| `extends`          | Keyword used for inheritance              |
| Single Inheritance | Only one parent class is inherited        |

---

## üîπ What is Multiple Inheritance?

**Multiple Inheritance** means:

> A class **inherits from more than one class**.

---

### ‚ùå Java Does **NOT** Support Multiple Inheritance With Classes

Java does **not** allow a class to inherit from **more than one class directly** using `extends`.

This is because:

> It causes **ambiguity** (confusion) when two parent classes have the **same method name**.

---

### ‚úÖ So, How to Do Multiple Inheritance in Java?

Java supports **multiple inheritance through interfaces** (not classes).

---

### ‚úÖ Real-Life Example:

* Suppose you have:

  * `Camera` ‚Üí takes photos
  * `Phone` ‚Üí makes calls
* You want to make a **SmartPhone** that does **both**.

This is possible using interfaces.

---

## ‚úÖ Java Code Example:

```java
// Interface 1
interface Camera {
    void takePhoto();
}

// Interface 2
interface Phone {
    void makeCall();
}

// Class implementing both interfaces
class SmartPhone implements Camera, Phone {
    public void takePhoto() {
        System.out.println("Taking a photo...");
    }

    public void makeCall() {
        System.out.println("Making a call...");
    }
}

// Main class
public class Main {
    public static void main(String[] args) {
        SmartPhone sp = new SmartPhone();
        sp.takePhoto();  // from Camera
        sp.makeCall();   // from Phone
    }
}
```

---

### üìå Output:

```
Taking a photo...
Making a call...
```

---

## üß† Why Interfaces?

Interfaces solve the **diamond problem** (method conflict from two parent classes).
Each interface is like a **contract** the child class agrees to follow.

---

## üéØ Summary:

| Concept                  | Inheritance Type        | Java Supports? | How to do it?           |
| ------------------------ | ----------------------- | -------------- | ----------------------- |
| **Single Inheritance**   | One parent class        | ‚úÖ Yes          | Using `extends`         |
| **Multiple Inheritance** | Multiple parent classes | ‚ùå No           | ‚úÖ Only via `interfaces` |
